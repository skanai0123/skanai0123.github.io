Overview

Use this file as the starting point for new subpages. Duplicate it into a new folder as index.html, then update the title, description, and body content.

The top-left shared panel automatically shows Back to Hub, Report Bug, and the metadata from the data-page-* attributes on <body>.

Starter Checklist

1. Update <title> and the description meta tag.

2. Pick a favicon from ../shared/icons/ that matches the topic instead of leaving the default blindly.

3. Add data-page-version or data-page-updated when you want to override the site-wide values.

4. Replace the hero text and add your page sections.

Version Policy

The shared header displays versions as v x.x.x.

Default auto behavior: use the latest Git tag when available; otherwise fall back to 0.0.0.

Use data-page-updated to pin the last substantive content update when metadata-only edits should not change the displayed date.

When creating a new page or making a larger update, ask whether the site should stay on the current major.minor line or bump to the next one.

Examples: keep 0.1.x for small additions, bump to 0.2.x for a new feature batch, bump to 1.0.x for a major milestone.
